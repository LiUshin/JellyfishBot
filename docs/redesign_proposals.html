JellyfishBot Settings Redesign

三套重设计方案：从「功能驱动」转向「体验与效能驱动」

方案一：极简效能版 (The Streamliner)
Focus: Speed

核心设计理念

消除所有不必要的层级，将 Master-Detail 结构优化为「即时响应式」。

  • Service 页：取消 Tab 切换，采用单页滚动 + 侧边锚点导航。
  • Scheduler 页：将运行记录从详情中分离，提供全局「任务监控面板」。
  • 交互：所有编辑均为 Inline（内联），点击文字直接修改，无需 Modal。

关键改进

针对高频操作进行路径缩短。

  • 列表卡片增加快速开关（Toggle）和一键运行按钮。
  • 能力配置改用图标网格，支持拖拽排序。
  • API Key 生成流程简化为单行操作。
方案二：数据仪表盘版 (The Dashboarder)
Focus: Insight

核心设计理念

将设置页转化为「控制中心」，强调数据反馈与状态监控。

  • Service 页：顶部增加统计卡片（活跃会话、Token 消耗、成功率）。
  • Scheduler 页：增加「运行日历」视图，直观查看任务分布与失败热点。
  • 视觉：使用更多渐变色和发光效果（Glow Effect）标记活跃状态。

关键改进

增强状态感知与异常处理。

  • WeChat 渠道增加「实时日志流」预览。
  • 任务运行记录支持按错误类型聚合。
  • 详情页增加「健康度」评分。
方案三：模块化向导版 (The Architect)

核心设计理念

针对复杂配置提供清晰的模块化路径，降低认知负担。

  • Service 页：采用「模块堆叠」布局，每个功能（WeChat, API, Docs）都是独立可拆卸的卡片。
  • Scheduler 页：创建流程改为分步向导（Wizard），引导用户完成复杂的权限设置。
  • 视觉：强调容器感，使用深浅交替的背景层级区分功能块。

关键改进

提升配置的严谨性与可维护性。

  • 权限设置（Sandbox）增加可视化路径选择器。
  • 支持「配置模板」，可快速复用 Service 设置。
  • Test Tab 升级为完整的「Playground」沙盒测试环境。
方案对比总结
维度 方案一：极简效能 方案二：数据仪表盘 方案三：模块化向导
适用人群 资深开发者 / 极客 运营 / 管理人员 企业级用户 / 新手
Service 详情 单页锚点滚动 带统计概览的 Tabs 可折叠模块卡片
任务运行记录 侧边抽屉弹出 日历 + 聚合列表 嵌入式时间轴
创建体验 快速内联创建 标准 Modal 分步向导 (Wizard)
视觉风格 高对比度、紧凑 发光特效、数据可视化 层次分明、模块化

专家建议：混合方案 (The Jellyfish Hybrid)

基于 JellyfishBot 的实际使用场景，我们建议采用以下混合策略：

  • 布局：采用方案三的「模块化卡片」，但保留方案一的「侧边锚点」以便快速跳转。
  • 统计：在 Header 引入方案二的「关键指标小卡片」，让状态一目了然。
  • 创建：使用方案三的「分步向导」，但在熟练后提供「快速创建」入口。
  • 测试：强烈建议将 Test Tab 升级为 Playground，直接在设置页进行对话调试。
